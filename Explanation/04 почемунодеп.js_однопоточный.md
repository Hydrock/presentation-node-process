# Слайд 4: Почему Node.js однопоточный

Этот слайд объясняет причины однопоточности Node.js и её преимущества.

Причины однопоточности Node.js:
- Облегченная модель выполнения без сложности многопоточности
- Избежание гонок данных (race conditions) при работе с общими ресурсами
- Простота отладки и понимания логики выполнения
- Эффективное управление ресурсами за счет Event Loop
- Одна точка входа в событийный цикл упрощает архитектуру

Преимущества:
- Нет необходимости в сложной синхронизации между потоками
- Упрощённый код без блокировок и мьютексов
- Меньше возможностей для ошибок из-за конкурентного доступа
- Эффективное использование ресурсов для I/O операций
- Хорошо подходит для I/O-интенсивных приложений

Недостатки:
- CPU-интенсивные задачи блокируют основной поток
- Необходимость использования Worker Threads для тяжелых вычислений
- Ограниченная эффективность при параллельных вычислениях
