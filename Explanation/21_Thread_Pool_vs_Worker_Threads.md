# Слайд 21: Thread Pool vs Worker Threads

Этот слайд сравнивает два разных механизма многопоточности в Node.js.

Thread Pool:
- Скрытый механизм внутри libuv
- Используется автоматически инфраструктурой
- Для блокирующих операций
- По умолчанию 4 потока
- Не требует явного создания
- Ограниченные возможности управления
- Операции автоматически распределяются
- Скрыт от программиста

Worker Threads:
- Явно создаваемые программистом
- Для CPU-интенсивных задач
- Собственный JS runtime/V8
- Изоляция кода и памяти
- Полный контроль над потоками
- Возможность создания произвольного количества
- Собственные API и методы
- Непосредственное управление

Ключевые различия:

1. Управление:
   - Thread Pool: автоматическое управление
   - Worker Threads: явное управление

2. Назначение:
   - Thread Pool: блокирующие операции
   - Worker Threads: CPU-интенсивные задачи

3. Использование:
   - Thread Pool: скрытый механизм
   - Worker Threads: явное использование

4. Ресурсы:
   - Thread Pool: ограниченное количество потоков
   - Worker Threads: произвольное количество

5. Изоляция:
   - Thread Pool: общая память
   - Worker Threads: изолированная память

Примеры использования:
- Thread Pool: криптографические операции, сжатие
- Worker Threads: математические вычисления, обработка данных
