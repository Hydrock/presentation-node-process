# Слайд 7: Libuv и потоки

Этот слайд подробно объясняет роль Libuv в многопоточности Node.js.

Libuv - это кроссплатформенная библиотека для асинхронного ввода-вывода:

Основные функции Libuv:
- Обеспечивает Event Loop для управления асинхронными операциями
- Управляет потоками (thread pool) для выполнения блокирующих операций
- Работает с системными вызовами операционной системы
- Предоставляет абстракцию для разных ОС (Linux, Windows, macOS)

Thread Pool Libuv:
- По умолчанию содержит 4 потока
- Может быть настроен через переменную окружения UV_THREADPOOL_SIZE
- Используется для операций, которые могут блокировать основной поток
- Обрабатывает CPU-интенсивные операции, которые не могут быть асинхронными

Пример настройки:
```bash
export UV_THREADPOOL_SIZE=8
node app.js
```

Ключевые особенности:
- Thread Pool управляется автоматически Libuv
- Операции помещаются в очередь при превышении количества потоков
- Каждый поток может обрабатывать только одну задачу за раз
- Thread Pool предназначен для блокирующих операций, не для CPU-интенсивных задач
