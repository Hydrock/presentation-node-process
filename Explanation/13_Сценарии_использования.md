# Слайд 13: Сценарии использования

Этот слайд объясняет конкретные сценарии применения различных подходов к многопоточности.

Используйте Event Loop когда:
- Файловые операции (fs.readFile, fs.writeFile)
- Сетевые вызовы (http, https)
- Работа с базами данных
- Таймеры и задержки
- Обработка событий пользовательского интерфейса
- Все I/O операции

Используйте Worker Threads когда:
- Математические вычисления (crypto, численные методы)
- Обработка больших объемов данных
- Шифрование/дешифрование
- Работа с изображениями
- Аналитика логов
- Парсинг больших файлов
- Обработка медиа-данных
- CPU-интенсивные задачи

Используйте Thread Pool когда:
- Криптографические операции (хэширование, шифрование)
- Сжатие данных (zlib)
- Часть файловых операций (fs)
- Некоторые DNS-запросы
- Блокирующие операции, которые не могут быть асинхронными

Ключевые принципы:
1. I/O операции - Event Loop
2. CPU операции - Worker Threads
3. Блокирующие операции - Thread Pool
4. Ограничьте количество потоков
5. Следите за производительностью
6. Используйте оптимальные размеры пулов

Примеры:
- Счетчики в Event Loop - быстрое выполнение
- Обработка изображений в Worker - параллельная обработка
- Сетевые запросы - асинхронные вызовы
- Математические расчеты - CPU интенсивные задачи
- Хэширование - через Thread Pool
