# Слайд 18: Thread Pool

Этот слайд объясняет концепцию Thread Pool в Node.js.

Thread Pool - это набор заранее созданных системных потоков внутри одного процесса Node.js:

Основные характеристики:
- Набор из 4 потоков по умолчанию (настраиваемый через UV_THREADPOOL_SIZE)
- Скрыт внутри библиотеки libuv
- Используется для выполнения блокирующих операций
- Все потоки находятся внутри одного процесса Node.js
- Не являются отдельными процессами, а именно нативными потоками

Использование:
- Для операций, которые не могут быть выполнены асинхронно через неблокирующие системные вызовы
- Автоматически управляется libuv
- Программист не создает их явно
- Операции помещаются в очередь при превышении количества потоков

Пример из презентации:
```javascript
const crypto = require('crypto');

// Эти операции используют thread pool
crypto.pbkdf2('password', 'salt', 100000, 64, 'sha512', (err, key) => {
  console.log(key.toString('hex'));
});
```

Ключевые особенности:
- Не требует явного управления
- Скрытый механизм для упрощения разработки
- Оптимизирован для частых операций
- Обеспечивает эффективное использование ресурсов
