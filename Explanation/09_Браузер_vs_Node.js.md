# Слайд 9: Браузер vs Node.js

Этот слайд сравнивает подходы к многопоточности в браузере и Node.js.

Браузер:
- Один поток для UI (рендеринг, события)
- Web Workers для параллелизма
- Ограниченный доступ к системным ресурсам
- Ограничения безопасности (CORS, Same-Origin)
- Ограниченная возможность взаимодействия с файловой системой

Node.js:
- Один поток для JavaScript кода
- Worker Threads для CPU-интенсивных задач
- Полный доступ к системным ресурсам
- Многопоточность через Libuv
- Меньше ограничений безопасности

Ключевые различия:
1. Потоки в браузере ограничены для безопасности
2. Node.js предоставляет больше возможностей через системные вызовы
3. Браузер использует Web Workers для параллелизма
4. Node.js использует Worker Threads и Libuv
5. Ограничения доступа к ресурсам
