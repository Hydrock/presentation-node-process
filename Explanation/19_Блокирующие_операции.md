# Слайд 19: Блокирующие операции

Этот слайд описывает типы операций, которые требуют использования Thread Pool.

Блокирующие операции - это операции, которые не могут быть выполнены асинхронно через неблокирующие системные вызовы:

Примеры блокирующих операций:
1. Хэширование:
   - bcrypt
   - PBKDF2
   - crypto-хэширование

2. Сжатие данных:
   - zlib (сжатие/распаковка)

3. Файловые операции:
   - Часть операций fs (например, fs.readFileSync)

4. DNS-запросы:
   - Некоторые DNS-запросы, которые требуют синхронного выполнения

5. Обработка больших объемов данных:
   - Парсинг больших JSON файлов
   - Обработка медиа-данных

Почему они блокирующие:
- Не имеют неблокирующих аналогов в системных вызовах
- Требуют значительных вычислительных ресурсов
- Могут занять значительное время выполнения
- Не могут быть выполнены в асинхронном режиме

Важно:
- Эти операции передаются в Thread Pool для выполнения
- Основной поток Node.js не блокируется
- Результат возвращается обратно через Event Loop
- Потенциальная угроза перегрузки Thread Pool при частом использовании
