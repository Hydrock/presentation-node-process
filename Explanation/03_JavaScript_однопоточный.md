# Слайд 3: JavaScript однопоточный

Этот слайд объясняет почему JavaScript является однопоточным языком и как это влияет на Node.js.

Основные характеристики однопоточности JavaScript:
- Один поток выполнения для выполнения кода
- Событийно-ориентированная модель (Event-driven)
- Асинхронное выполнение операций
- Одновременно только один фрагмент кода выполняется

Важные аспекты:
- В браузере однопоточность ограничивает выполнение JavaScript
- В Node.js однопоточность позволяет эффективно обрабатывать I/O операции
- Event Loop управляет очередью асинхронных операций
- Код в основном потоке не блокируется при выполнении I/O

Пример из презентации:
```javascript
console.log("1");
setTimeout(() => console.log("2"), 0);
console.log("3");
// Вывод: 1, 3, 2 (не 1, 2, 3)
```

Это демонстрирует асинхронное выполнение, где setTimeout помещается в очередь и выполняется позже, когда основной поток освобождается.
